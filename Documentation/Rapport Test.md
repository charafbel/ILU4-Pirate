# Plan général

## Classes à tester

Selon les exigences fournie, les classes à tester intégralement seront celles des cases suivantes :

- Bombe
- Équipement
- Piège à ressort

## Tests à effectuer

Afin de tester correctement les classes, nous appliqueront avec rigueur les étapes suivantes :

1. **Liste des exigences**
2. **Plan de tests fonctionnels** :
    1. Classes d’équivalence
    2. Tests aux limites
3. **Test de couverture** :
    1. Couverture des instructions
    2. Couverture des branches
4. **Complétion de la liste de tests ?** (Si les couvertures ne sont pas à 100%)
5. **Test de l’exécution** (Vérifier si la fonction fait ce qu’il faut)
6. **Analyse des défauts ?** (Si tous les tests ne sont pas validés) → Rectifications du code
7. **Création des mutants**
8. **Analyse des mutants survivants ?** (S’il y en a)
9. **Modification de la liste de tests ?** (S’il existe des mutants survivant et non-équivalents → notre liste de tests et incomplète) → Une fois la liste complétée, on repart à partir du **Test de l’exécution**.
